KEY CHANGES FROM CURRENT STRUCTURE:
===================================

1. controllers/ → viewmodels/          (Rename to reflect MVVM pattern)
2. screens/ → views/                   (Rename to reflect MVVM pattern)
3. utils/ → core/utils/                (Move to core for better organization)
4. common/ → shared/                   (Rename for clarity)
5. All feature modules follow: models/ | viewmodels/ | views/ | bindings/


MVVM LAYER RESPONSIBILITIES:
============================

MODEL:
  - Pure data classes
  - JSON serialization/deserialization
  - No business logic
  - No UI dependencies

VIEWMODEL:
  - State management (Rx observables)
  - Business logic
  - Repository communication
  - Data transformation for views
  - User action handling

VIEW:
  - UI rendering
  - User interaction handling
  - Display data from ViewModel
  - Minimal state (only UI state)

REPOSITORY:
  - Data access abstraction
  - API/database communication
  - Data caching
  - No business logic
